# Лабораторная работа 1
## Знакомство с клиент-серверным взаимодействием

### Цель работы
Изучение основ клиент-серверного взаимодействия в веб-приложениях, освоение базовых принципов работы с HTTP-протоколом, получение практических навыков создания простого веб-сервера и обработки HTTP-запросов с использованием различных технологических стеков.

### Технические требования:
- Наличие интернет-соединения
- Наличие [cURL](https://curl.se/download.html) / [Postman](https://www.postman.com/downloads/) / [Insomnia](https://insomnia.rest/download)
- Наличие [Docker](https://docs.docker.com/desktop/) и [Docker Compose](https://docs.docker.com/compose/install/)
- Наличие настроенного окружения для работы с выбранным языком программирования (интерпретатор, компилятор, менеджер зависимостей [при необходимости])

### Технические ограничения:
Допускается использование следующих языков программирования и HTTP-фреймворков:
- TypeScript / [NestJS](https://nestjs.com/)
- Java / [Spring Framework](https://spring.io/) / [Spring Boot](https://spring.io/projects/spring-boot)
- Python / [Django](https://www.djangoproject.com/) / [Flask](https://flask.palletsprojects.com/) / [FastAPI](https://fastapi.tiangolo.com/)
- Go / [Gin](https://gin-gonic.com/) / [Fiber](https://gofiber.io/) / [Echo](https://echo.labstack.com/) / [Beego](https://beego.vip/)
- PHP / [Laravel](https://laravel.com/)

### Ход работы:
#### 1. Создание проекта
Создайте новый проект с использованием выбранного языка программирования и фреймворка. Инициализируйте необходимые конфигурационные файлы и зависимости.

#### 2. Реализация обработчика запросов
Модифицируйте исходный код, добавив возможность обработки HTTP GET-запросов по произвольному маршруту (например, `GET /info`). Пример запроса с использованием cURL представлен ниже:
   ```bash
   curl --location 'http://localhost:4200/info'
   ```

#### 3. Реализация бизнес-логики
Модифицируйте исходный код, добавив возможность отправки ответа на запрос `GET /info`, содержащего JSON с полем, отражающим количество дней до наступления нового года. Допустимо использование стандартных библиотек для работы с датами. Избегайте использования библиотек, не поддерживаемых разработчиками фреймворка.

   Пример ответа на запрос:
   ```json
   {
       "days_before_new_year": 123
   }
   ```

#### 4. Контейнеризация приложения
Создайте `Dockerfile` для возможности запуска проекта на устройствах, не имеющих установленного компилятора / интерпретатора для запуска кода. Допускается добавление `docker-compose.yml` файла для упрощения запуска приложения.

#### 5. Документация
Создайте `README.md` файл в проекте с описанием основных команд для запуска приложения, включая инструкции по сборке и запуску контейнера.

### Критерии приемки
1. Репозиторий с кодом на GitHub
2. README.md файл с описанием основных команд для запуска приложения в виде Docker-контейнера
3. Рабочий пример приложения, обрабатывающий запросы по указанному маршруту
4. Корректный расчет количества дней до нового года
